# Справочник конфигурации

В этом документе описаны все переменные окружения времени выполнения, тома,
открытые порты и аргументы сборки образа `6run0/angie`.

См. также: [../README.md](../README.md)

---

## Переменные окружения

Все тумблеры принимают одинаковый набор истинных значений:
`yes`, `on`, `1`, `true`, `enable`, `enabled`.
Любое другое значение (включая отсутствие переменной) трактуется как отключено.

Переменные-тумблеры не дают эффекта, если образ запущен с `docker run --user`
под UID, который не является владельцем директорий конфигурации Angie
(`/etc/angie/http-conf.d`, `/etc/angie/modules.d`,
`/etc/angie/http-conf-available.d`). В таком случае точка входа выводит
предупреждение и использует конфигурацию, запечённую на этапе сборки.
Непривилегированный образ (`-unprivileged`) заблаговременно передаёт права на
эти директории своему UID по умолчанию, поэтому переключение в рантайме
работает без дополнительных настроек.

### Общие

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANGIE_ENTRYPOINT_QUIET_LOGS` | не задана | Подавить информационные сообщения точки входа. Выводятся только предупреждения и ошибки. Задайте любое непустое значение для активации. |
| `ANGIE_ENTRYPOINT_WORKER_PROCESSES_AUTOTUNE` | не задана | Автоматически настраивать `worker_processes` по числу CPU в рантайме (читает квоту и cpuset из cgroup v1/v2, берёт минимум). Не работает, если `/etc/angie/angie.conf` смонтирован только для чтения. Задайте любое непустое значение для активации. |

### Сжатие

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANGIE_BROTLI_ENABLED` | `no` | Загрузить модуль Brotli filter и включить сжатие Brotli на лету. |
| `ANGIE_BROTLI_STATIC_ENABLED` | `no` | Включить сжатие Brotli и отдачу предварительно сжатых файлов `*.br`. Подразумевает `ANGIE_BROTLI_ENABLED`. |
| `ANGIE_GZIP_ENABLED` | `no` | Включить сжатие gzip на лету. |
| `ANGIE_GZIP_STATIC_ENABLED` | `no` | Включить сжатие gzip и отдачу предварительно сжатых файлов `*.gz`. Подразумевает `ANGIE_GZIP_ENABLED`. |
| `ANGIE_ZSTD_ENABLED` | `no` | Загрузить модуль Zstandard filter и включить сжатие zstd на лету (`zstd_comp_level 1`; поднимите уровень по месту через пользовательский том). |
| `ANGIE_ZSTD_STATIC_ENABLED` | `no` | Включить сжатие zstd и отдачу предварительно сжатых файлов `*.zst`. Подразумевает `ANGIE_ZSTD_ENABLED`. |

> **Примечание о кэшировании:** заголовок `Vary: Accept-Encoding` ставит только
> gzip (директивой `gzip_vary on`); brotli и zstd сами по себе его **не**
> выставляют. Пока gzip включён, он добавляет `Vary` к любому ответу, включая
> сжатые другим кодеком. Если включить brotli или zstd **без** gzip за общим или
> CDN-кэшем, кэш может выдать сжатое тело клиенту, который такую кодировку не
> запрашивал -- держите gzip включённым или добавьте `Vary: Accept-Encoding` на
> уровне vhost. `ANGIE_ZSTD_STATIC_ENABLED` отдаёт соседний `*.zst`, если он
> есть и клиент принимает zstd, иначе молча возвращается к сжатию на лету --
> отсутствие `*.zst` не является ошибкой.

### Динамические модули

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANGIE_MODSECURITY_ENABLED` | `no` | Загрузить модуль ModSecurity (только движок WAF; правила в образ не включены -- смонтируйте набор правил через пользовательский том). |
| `ANGIE_SUBS_ENABLED` | `no` | Загрузить модуль HTTP substitutions filter. |

### GeoIP2

| Переменная | По умолчанию | Описание |
|---|---|---|
| `GEOIP2_DB_COUNTRY` | не задана | Абсолютный путь внутри контейнера до файла базы данных MaxMind GeoIP2 (страны). Если задан и файл доступен для чтения, модуль GeoIP2 и соответствующий конфигурационный сниппет включаются автоматически. Путь ограничен символами `A-Za-z0-9._/-`; любой другой символ вызывает ошибку для предотвращения инъекции в конфигурацию. |

### Форматы логов

Переменные "register" добавляют определение формата в конфигурацию Angie, чтобы
на него можно было ссылаться, но не изменяют то, что пишется в `/dev/stdout`.
Переменные "use" одновременно регистрируют формат и активируют его как лог
доступа на `/dev/stdout`. Одновременно должна быть включена только одна
переменная "use" -- при нескольких активных каждый запрос логируется несколько раз.

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANGIE_LOG_FORMAT_EXTENDED` | `no` | Зарегистрировать формат `extended` (расширенные поля). Активный лог не меняет. |
| `ANGIE_LOG_FORMAT_LOGFMT` | `no` | Зарегистрировать формат `logfmt` (key=value). Активный лог не меняет. |
| `ANGIE_LOG_FORMAT_MAIN` | `no` | Зарегистрировать классический формат `main`. Активный лог не меняет. |
| `ANGIE_LOG_FORMAT_MATOMO` | `no` | Зарегистрировать формат `matomo` (аналитика Matomo). Активный лог не меняет. |
| `ANGIE_LOG_EXTENDED` | `no` | Зарегистрировать и использовать формат `extended` для `/dev/stdout`. |
| `ANGIE_LOG_LOGFMT` | `yes` | Зарегистрировать и использовать формат `logfmt` для `/dev/stdout`. Включён по умолчанию -- стандартный лог доступа ведётся в формате logfmt. |
| `ANGIE_LOG_MAIN` | `no` | Зарегистрировать и использовать формат `main` для `/dev/stdout`. |
| `ANGIE_LOG_MATOMO` | `no` | Зарегистрировать и использовать формат `matomo` для `/dev/stdout`. |
| `ANGIE_LOG_FORMAT_LOGFMT_GEOIP2` | `no` | Требует активного GeoIP2 (переменная `GEOIP2_DB_COUNTRY` задана и файл читаем). Зарегистрировать формат `logfmt-with-geoip2` (добавляет поле `country`) без активации. |
| `ANGIE_LOG_LOGFMT_GEOIP2` | `no` | Требует активного GeoIP2 (переменная `GEOIP2_DB_COUNTRY` задана и файл читаем). Зарегистрировать и использовать формат `logfmt-with-geoip2` для `/dev/stdout`. |

### Real IP

Восстановление реального адреса клиента, когда образ работает за доверенным
прокси, балансировщиком нагрузки или ingress (используется встроенный модуль
`ngx_http_realip_module`; пакет не нужен). Включается по наличию: установка
`ANGIE_REAL_IP_FROM` активирует функцию.

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANGIE_REAL_IP_FROM` | не задана | Список доверенных адресов/CIDR прокси (IPv4 или IPv6) через пробел или запятую. Если задан, сниппет real-IP рендерится и включается. Каждый элемент ограничен символами `0-9 a-f A-F : . /`; любой другой символ вызывает ошибку для предотвращения инъекции в конфигурацию. |
| `ANGIE_REAL_IP_HEADER` | `X-Forwarded-For` | Заголовок (или ключевое слово `proxy_protocol`), из которого читается реальный адрес клиента. Ограничен символами `A-Za-z0-9_-`. |
| `ANGIE_REAL_IP_RECURSIVE` | `on` | `on` идёт по цепочке `X-Forwarded-For` справа налево, пропуская доверенные адреса, чтобы найти реального клиента за несколькими прокси; `off` берёт последний (правый) адрес в заголовке — тот, что вставил ближайший доверенный прокси. Допустимо только `on` или `off`. |

> **Безопасность:** перечисляйте **только** действительно доверенные прокси.
> Шаблон вроде `0.0.0.0/0` позволит любому клиенту подделать выбранный заголовок
> и подменить `$remote_addr`, что также обходит loopback-защиту эндпоинта
> `/healthz`. Сам `HEALTHCHECK` контейнера не затрагивается: он подключается к
> `127.0.0.1` без заголовка `X-Forwarded-For`, поэтому real-IP ничего не
> переписывает и `$remote_addr` остаётся loopback-адресом.
>
> **Примечание:** `ANGIE_REAL_IP_HEADER=proxy_protocol` дополнительно требует,
> чтобы слушатель работал в режиме PROXY-протокола (`listen ... proxy_protocol`),
> чего запечённый слушатель `:80` (или `:8080` в unprivileged) не делает.
> Добавьте свой vhost со слушателем `proxy_protocol` через `/etc/angie/custom`,
> чтобы использовать этот режим.

### Заголовки безопасности

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANGIE_SECURITY_HEADERS_ENABLED` | `no` | Отдавать консервативный базовый набор заголовков ответа (с `always`, поэтому они применяются и к ответам с ошибками): `X-Content-Type-Options: nosniff`, `Referrer-Policy: strict-origin-when-cross-origin`, `X-Frame-Options: SAMEORIGIN`, `Permissions-Policy: camera=(), microphone=(), geolocation=()`. HSTS, CSP и устаревший `X-XSS-Protection` намеренно не включены -- добавьте их по месту через пользовательский том. |

> **Примечание:** `add_header` **не** объединяется между контекстами. Блок
> `server` или `location`, объявляющий собственный `add_header`, заменяет
> унаследованный набор, отбрасывая эти заголовки. Если пользовательский vhost
> добавляет заголовки, повторите в нём и базовые.

### Файловая система

| Переменная | По умолчанию | Описание |
|---|---|---|
| `CACHE_DIR` | не задана | Если задан путь к существующей директории, точка входа рекурсивно меняет её владельца на пользователя `angie` при старте. Полезно при монтировании директории кэша прокси через bind mount. Выполняется только при запуске от root (не работает в непривилегированном образе). |

## Зависимости между переменными

- `ANGIE_BROTLI_STATIC_ENABLED` активирует базовый тумблер Brotli
  (`ANGIE_BROTLI_ENABLED`) перед включением статического модуля. Достаточно
  задать только переменную static.
- `ANGIE_GZIP_STATIC_ENABLED` активирует базовый тумблер gzip
  (`ANGIE_GZIP_ENABLED`) перед включением static. Достаточно задать только
  переменную static.
- `ANGIE_ZSTD_STATIC_ENABLED` активирует базовый тумблер zstd
  (`ANGIE_ZSTD_ENABLED`) перед включением статического модуля. Достаточно
  задать только переменную static.
- `ANGIE_REAL_IP_HEADER` и `ANGIE_REAL_IP_RECURSIVE` обрабатываются скриптом
  точки входа real-IP (`35-real-ip.sh`) и не действуют, если не задана
  `ANGIE_REAL_IP_FROM`.
- Переменные `ANGIE_LOG_FORMAT_*` только регистрируют определение формата. Они
  не выбирают активный лог доступа. Для регистрации и активации формата
  используйте переменные `ANGIE_LOG_*`.
- `ANGIE_LOG_LOGFMT_GEOIP2` и `ANGIE_LOG_FORMAT_LOGFMT_GEOIP2` обрабатываются
  скриптом точки входа GeoIP2 (`50-geoip2.sh`) и молча игнорируются, если
  `GEOIP2_DB_COUNTRY` не задана или файл базы данных недоступен для чтения.
- При установке переменной "use" для лога точка входа сначала отключает все
  остальные активные сниппеты лога доступа, гарантируя единственную директиву
  `access_log`.

## Тома

### `/etc/angie/custom`

Основная точка расширения. Смонтируйте директорию сюда, чтобы добавить
дополнительную конфигурацию Angie поверх встроенных значений по умолчанию без
редактирования образа:

```text
docker run -v /host/angie/custom:/etc/angie/custom:ro 6run0/angie
```

Директория включается из `angie.conf` после всех встроенных включений. Здесь
можно размещать серверные блоки, дополнительные сниппеты контекста `http {}` и
TLS-сертификаты. Монтирование в режиме только для чтения (`:ro`) рекомендуется,
когда контейнер не записывает конфигурацию в рантайме.

### `CACHE_DIR` (bind-mounted кэш)

При использовании Angie как кэширующего прокси смонтируйте директорию на хосте
для кэша и передайте её путь через `CACHE_DIR`, чтобы точка входа исправила
права владельца:

```text
docker run -v /host/cache:/var/cache/angie \
  -e CACHE_DIR=/var/cache/angie \
  6run0/angie
```

## Порты

| Вариант | Порт | Примечания |
|---|---|---|
| Стандартный (`6run0/angie`, `6run0/angie:latest`) | `80` | Требует root или `CAP_NET_BIND_SERVICE`. |
| Непривилегированный (`6run0/angie:...-unprivileged`) | `8080` | Доступен без привилегий. |

Эндпоинт проверки работоспособности доступен по `GET /healthz` и возвращает `ok`.
Запросы к нераспознанным виртуальным хостам возвращают `444` (соединение
закрывается без ответа).

## Аргументы сборки

Эти аргументы передаются в `docker build` через `--build-arg`. Полные примеры
сборки см. в [usage.md](usage.md).

### Версия Angie и метаданные образа (оба Dockerfile)

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `ANGIE_VERSION` | текущий пин | Устанавливаемая версия Angie (ядро + все модули), запинена через оператор `=~` в apk и резолв `madison` в apt — образ воспроизводим. Release-workflow подставляет её из git-тега и отказывается публиковать, если она расходится с пином. |
| `IMAGE_VERSION` | `dev` | Версия упаковки в метке `org.opencontainers.image.version` (тег образа без суффикса варианта, например `1.11.8-build1`). Задаётся release-workflow. |
| `VCS_REF` | пусто | Коммит источника в метке `org.opencontainers.image.revision`. Задаётся release-workflow. |

Образ несёт OCI-метки: набор `org.opencontainers.image.*` (`title`,
`description`, `source`, `url`, `documentation`, `licenses`, `version`,
`revision`) плюс `software.angie.version` (точная версия Angie). Читаются через
`docker inspect --format '{{json .Config.Labels}}' <image>`.

### Идентификатор пользователя непривилегированного образа (`Dockerfile.unprivileged`)

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `APP_USER` | `app` | Имя пользователя ОС, создаваемого внутри образа. |
| `APP_GROUP` | `app` | Имя группы ОС. |
| `APP_UID` | `65532` | UID непривилегированного пользователя (совпадает с "nonroot" в distroless). |
| `APP_GID` | `65532` | GID непривилегированной группы. |

### Только для Debian (`debian/Dockerfile`)

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `DEBIAN_MIRROR` | значение по умолчанию | URL зеркала APT для пакетов Debian. Укажите внутреннее зеркало при необходимости. |
| `DEBIAN_SECURITY_MIRROR` | значение по умолчанию | URL зеркала APT для обновлений безопасности Debian. |

### Пин angie-ctl (оба Dockerfile)

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `ANGIE_CTL_COMMIT` | зафиксированный SHA | Хеш коммита Git вспомогательного инструмента [angie-ctl](https://github.com/6RUN0/angie-ctl), клонируемого при сборке. Укажите другой хеш для проверки новой версии. |
