# История изменений

Здесь документируются все значимые изменения образа.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).
Образы версионируются по релизу вышестоящего Angie плюс номер сборки
упаковки — `<angie>-build<N>` — а не по семантическому версионированию самой
упаковки. Номер сборки увеличивается, когда та же версия Angie переупаковывается
(обновление базового образа, исправление entrypoint, обновление `angie-ctl`).

## [Не выпущено]

### Добавлено

- **Сжатие Zstandard** — пятый встроенный динамический модуль
  (`angie-module-zstd`), поставляется выключенным и включается тумблерами
  `ANGIE_ZSTD_ENABLED` / `ANGIE_ZSTD_STATIC_ENABLED`, симметрично управлению
  Brotli и gzip.
- **Восстановление реального IP** за доверенным прокси/балансировщиком/ingress
  через `ANGIE_REAL_IP_FROM` (плюс `ANGIE_REAL_IP_HEADER` и
  `ANGIE_REAL_IP_RECURSIVE`), с проверкой набора символов каждого доверенного
  адреса для предотвращения инъекции в конфигурацию. Использует встроенный
  модуль real-IP — без дополнительного пакета.
- **Базовые security-заголовки ответа**, включаемые тумблером
  `ANGIE_SECURITY_HEADERS_ENABLED` (`X-Content-Type-Options`, `Referrer-Policy`,
  `X-Frame-Options`, `Permissions-Policy`), применяются с `always`.

## [1.11.8-build1] - 2026-06-23

Первый публичный релиз: Angie 1.11.8, собранный для `linux/amd64` и `linux/arm64`.

### Добавлено

- Angie 1.11.8 на двух базах — **Alpine** (по умолчанию) и **Debian** — каждая
  с непривилегированным (rootless) вариантом (uid/gid 65532, слушает порт 8080).
- Четыре встроенных динамических модуля, поставляемых **выключенными** и
  включаемых при старте контейнера через переменные окружения `ANGIE_*`: Brotli,
  GeoIP2, ModSecurity (WAF) и фильтр подстановок HTTP.
- Тумблеры функций времени выполнения по конвенции `ANGIE_*_ENABLED`, а также
  управление статическим сжатием, форматами логов, картой WebSocket и
  автоподстройкой числа рабочих процессов (полный справочник в
  [docs/configuration.ru.md](docs/configuration.ru.md)).
- Модель активации `available.d`/`.d`: сниппеты конфигурации и загрузки модулей
  поставляются выключенными и включаются символьными ссылками при старте через
  `angie-ctl`.
- Наложение пользовательской конфигурации через том `/etc/angie/custom` поверх
  каждого встроенного include без правки поставляемых файлов.
- Эндпоинт проверки живости `/healthz` (только loopback), связанный с
  `HEALTHCHECK` образа.
- Метки OCI-образа, включая `software.angie.version` с версией упакованного Angie.
- Публикация в GitHub Container Registry (`ghcr.io/6run0/docker-angie`) и
  Docker Hub (`6run0/angie`) с неизменяемыми тегами `…-build<N>` и плавающими тегами.
